# Objetivo del Proyecto

## 1. Nombre del proyecto

El proyecto se llamará **Mora Vinería**.

## 2. Descripción general

**Mora Vinería** es una aplicación web simple, útil y mobile-first, pensada para registrar ventas, registrar gastos y consultar reportes básicos de una vinería.

El objetivo principal de la app es ayudar a mantener el orden del negocio, centralizar la información diaria y facilitar una mejor organización comercial sin convertir el sistema en una herramienta compleja o difícil de usar.

La aplicación está pensada para una vinería pequeña, donde la operación diaria necesita ser rápida, clara y práctica.

La app debe contemplar que gran parte de la actividad comercial ocurre durante la noche y la madrugada, por lo que el diseño, la carga de ventas y los reportes deben adaptarse a una jornada de venta que no necesariamente coincide con el día calendario tradicional.

## 3. Usuario principal

El usuario principal de la aplicación es la **dueña de la vinería**, quien también atiende el negocio y se encarga de la gestión diaria.

Puede haber ayudantes casuales, pero para la primera versión no se manejarán empleados, roles diferenciados ni permisos complejos. Todos los usuarios que utilicen la app tendrán el mismo nivel de responsabilidad operativa y acceso funcional.

Por lo tanto, la app debe diseñarse bajo la idea de un uso simple, directo y compartido, sin estructuras avanzadas de administración de personal.

## 4. Problema que resuelve

Actualmente, la gestión de ventas y gastos puede volverse desordenada cuando se registra de forma manual, informal o distribuida en distintos lugares.

La app busca resolver principalmente los siguientes problemas:

- No tener claridad sobre cuánto se vende por día, semana o mes.
- No saber con precisión cuánto se gasta.
- No poder estimar fácilmente cuánto queda como ganancia.
- No distinguir con claridad entre dinero vendido, dinero reinvertido y ganancia neta estimada.
- Tener ventas y gastos anotados de forma desordenada.
- No contar con reportes simples para entender el estado del negocio.
- No poder estimar cómo podría terminar el mes si se mantiene el ritmo actual de ventas y gastos.

La finalidad no es reemplazar una contabilidad formal, sino brindar una herramienta de control interno sencilla para tomar mejores decisiones cotidianas.

## 5. Objetivos principales

La aplicación debe permitir:

- Registrar ventas.
- Registrar gastos.
- Administrar productos.
- Consultar un resumen diario.
- Consultar un resumen semanal.
- Consultar un resumen mensual.
- Ver una ganancia estimada.
- Ver productos más vendidos.
- Ver medios de pago más utilizados.
- Ver una proyección mensual simple.

Estos objetivos deben mantenerse como guía principal durante el diseño y desarrollo del MVP.

## 6. Alcance inicial del MVP

La primera versión funcional de **Mora Vinería** debe incluir como mínimo:

- Gestión de productos.
- Registro de ventas.
- Registro de gastos.
- Dashboard inicial.
- Reportes simples.
- Proyecciones básicas.

El MVP debe priorizar la utilidad diaria por encima de la complejidad técnica. La app debe poder usarse desde el celular sin dificultad, permitiendo cargar información rápidamente y consultar datos relevantes del negocio en pocos pasos.

## 7. Fuera de alcance

Para evitar que el proyecto crezca de forma innecesaria, la primera versión de la app no debe incluir:

- Gestión de combos.
- Facturación fiscal.
- Múltiples sucursales.
- Contabilidad avanzada.
- Gestión de empleados.
- Roles o permisos complejos.
- Stock avanzado.
- Funciones propias de un ERP.

La app no debe transformarse en un sistema empresarial pesado. Cualquier funcionalidad nueva deberá evaluarse según su aporte real al objetivo principal del proyecto.

## 8. Principios del producto

La aplicación debe respetar los siguientes principios:

- Debe ser simple.
- Debe ser minimalista.
- Debe ser visualmente agradable.
- Debe ser rápida.
- Debe estar pensada primero para celular.
- Debe ser fácil de usar.
- No debe sentirse como un sistema empresarial pesado.
- No debe requerir conocimientos técnicos para operar.
- No debe agregar pasos innecesarios al registro de ventas o gastos.

Los detalles visuales específicos, como paleta de colores, componentes, estilo de interfaz, jerarquía visual y diseño de pantallas, se definirán en el documento de diseño de la app.

## 9. Criterio de éxito del MVP

La primera versión del sistema podrá considerarse útil cuando permita:

- Registrar ventas desde el celular sin complicaciones.
- Registrar gastos desde el celular sin complicaciones.
- Administrar productos básicos.
- Ver cuánto se vendió y cuánto se gastó en el día.
- Ver cuánto se vendió y cuánto se gastó en la semana.
- Ver cuánto se vendió y cuánto se gastó en el mes.
- Estimar si el negocio está dejando ganancia.
- Consultar una proyección simple de cierre mensual.

El MVP será exitoso si ayuda a ordenar la operación diaria de la vinería y permite entender mejor el movimiento económico del negocio.

## 10. Prioridad inicial

La prioridad inicial del proyecto será permitir:

- Administrar productos.
- Registrar ventas.

Estas dos funciones se consideran inseparables para la primera etapa, ya que el registro de ventas depende de contar previamente con productos cargados en el sistema.

Una vez que estas funciones estén correctamente diseñadas y funcionando, se avanzará con gastos, dashboard, reportes y proyecciones.
